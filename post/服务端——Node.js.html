Node.js 初体验

Node.js 注意点

Buffer

Buffer 中文译为 缓冲区，是类似于 Array 的 对象，用于表示固定长度的字节序列，换而言之，Buffer 就是一段固定长度的内存空间，用于处理二进制数据。

Buffer 特点

  1. 与数组不同，Buffer 大小固定且无法调整

  2. Buffer 性能比较好，可以直接对计算机内存进行操作

  3. 每个元素的大小为 1 字节 （8 byte = 1 字节）

Buffer 创建

  1. alloc 方法

创建

let buf = Buffer.alloc(10);

该行代码的含义就是创建一个 10 字节的 Buffer，这里面的 Buffer 是 node.js 的内置模块，在启动时，已经将这个模块加载进来了，所以不用手动导入就可以使用。alloc 单词的含义是分配的意思，它是 Buffer 里面的一个方法。

控制台打印 buf 变量

console.log(buf)

  1. allocUnsafe
let buf = Buffer.allocUnsafe(10)

它所创建的 Buffer 可能会包含旧的内存数据。

// let buf = Buffer.allocUnsafe(10)
let buf_2 = Buffer.allocUnsafe(10000)

console.log(buf_2)

对于这样的 Buffer 在使用的时候需要格外小心。但是使用这种方式去创建 Buffer，它的速度比 alloc 方式创建要快，因为它不需要做旧数据归零这一操作。

  1. from 方法
let buf_3 = Buffer.from('hello')
let buf_4 = Buffer.from([105,108,111,118,101,121,111,117])

该方法能够将字符串或数组转换为 Buffer。

console.log(buf_3)
console.log(buf_4.toString())

fs 模块

fs => file system ‘文件操作系统’

fs 模块作用

fs 模块可以实现与硬盘的交互，例如文件的创建、删除、重命名、移动，还有文件内容的写入、读取以及文件夹的相关操作

fs模块-写入文件

/**
* 新建一个文件：座右铭.txt，写入内容，三人行，则必有我师焉
**/
  1. 导入 fs 模块

语法: jsconst fs = require('fs')

require 是一个全局的函数，用来导入模块，（'fs'），这里面的 fs 是模块的名称，不能随意命名，必须写成 fs。

  1. 写入文件

语法: fs.writeFile(file， data[， options]， callback)

参数说明:

  • file 文件名
  • data 待写入的数据
  • options 选项设置 (可选)
  • callback 写入回调

示例代码:

// require 是 Node.js 环境中的 '全局' 变量，用来导入模块

const fs = require('fs')

// 将【三人行，则必有我师焉。】写入当前文件夹下的【座右铭.txt】文件中

fs.writeFile('./座右铭.txt', '三人行，则必有我师焉。', err => {
    // 如果写入失败，则回调函数调用时，会传入错误对象，如写入成功，会传入 null
    if (err) {
        console.log('写入失败')
        return
    }
    console.log('写入成功')
})

使用脚本来写入文件的原因:

比如我们的一个网站，我们希望自动记录用户访问的时间，并且将访问记录放到一个文件当中。使用脚本可以很方便实现自动化写入文件。

fs 工作的两种模式 异步与同步

异步写入文件

const fs = require('fs')
fs.writeFile('./座右铭.txt', '三人行，则必有我师焉。', err => {
    if (err) {
        console.log('写入失败')
        return
    }
    console.log('写入成功')
})

console.log(1 + 1)

写入成功

在执行上方代码时，主线程执行到 fs.writeFile('./座右铭.txt', '三人行，则必有我师焉。',err => {...}) 时，就会有一条 I/O 线程进行磁盘写入，I/O 线程写入结束，就把回调压入到任务队列中，最后等主线程将初始化代码执行完毕后，再去任务队列里将回调取出来执行。

同步写入文件

fs.writeFileSync('./data.txt', 'test')

当主线程执行到该行代码时，就停了，要等待 I/O 线程执行完磁盘写入功能后，主线程再执行接下来的代码。

fs模块-文件追加写入

appendFile 作用是在文件尾部追加内容，appendFile 语法与 writeFile 语法完全相同

语法:

fs.appendFile(file, data[, options], callback)
fs.appendFileSync(file， data[， options])

代码示例:

const fs = require('fs')

fs.appendFile('./座右铭.txt', ',则其善者而从之', err => {
    if (err) {
        console.log('写入失败')
        return
    }
    console.log('写入成功')
})

fs.appendFileSync('./座右铭.txt', '温故而知新,可以为老师/r/n')

fs模块-文件流式写入

语法:

fs.createWriteStream(path[， options])

参数说明:

  • path 文件路径
  • options 选项配置(可选)

返回值: object

代码示例: 

/** 
* 观书有感.txt
*/

// 1. 导入 fs
const fs = require('fs');

// 2. 创建写入流对象
const ws = fs.createWriteStream('./观书有感.txt');

// 3. write
ws.write('半亩方塘一舰开\r\n');
ws.write('天光云影共徘徊\r\n');
ws.write('问渠哪得清如许\r\n');
ws.write('为有源头活水来\r\n');

// 4. 关闭通道
ws.close()

文件写入应用场景

文件写入 在计算机中是一个非常常见的操作，下面的场景都用到了文件写入

  • 下载文件
  • 安装软件
  • 保存程序日志，如 Git
  • 编辑器保存文件
  • 视频录制

当 需要持久化保存数据 的时候，应该想到 文件写入
