对象拓展

对象拓展

咱先来看看最初是怎么写的 👉

<script>
    let obj = {
        name: "Alian",
        a: function () {

        },
        b: function () {

        }
    }
</script>

那咋样进行简写呢？

属性可以简写，方法也是可以简写的，在 Vue 中就是这么干滴!!! 在使用 Vue 的时候，进行模块化组件化开发的时候，是经常会给这个对象取取一个名字的，将来导出的时候非常方便，这样的简写方式是很有必要的。

代码示例 👉

<script>
    let name = "模块A"
    let obj = {
        name,  // name:name
        a() {

        },
        b() {

        }
    }
</script>

对象属性 – 表达式写法

咱先来看这个问题，这个obj里面的name属性是name还是变成 ‘a’了呢？

<script>
    let name = 'a'
    let obj = {
        name: '值'
    }
</script>

经过控制台打印发现还是 name，也就是说 obj 里面的 name 并没有被当作是变量来处理，那咱就是想要这个name变成a，咋办呢？巧了，ES6又给咱提供了方法[…..]，可以将里面的属性名(key)变成变量。而且[…..]里面是可以写表达式滴！

代码示例 👉

<script>
    let name = 'a'
    let name1 = 'b'
    let obj = {
        [name]: '值',
        [name1 + "cd"]: '值'
    }

    console.log(obj);
</script>

---控制台---
object
a:"值"
bcd:"值"
[[Prototype]]: object

提需求：咱现在希望这个模块A里面的每一个方法名字都得带上模块A

代码示例 👉

<script>
    let name = '模块A'
    let obj = {
        name,
        f1() {

        },
        f2() {

        }
    }
    console.log(obj)
</script>

咱现在就用这个 […] 的方法，这便是模块化思想

    name,
    [name + "f1"]() {

    },
    [name + "f2"]() {

    }

---控制台---
object
name:"模块A"
模块Af1: f [name +“f1"]()
棋块Af2: f [name +“f2"]()
[[Prototype]]: object

… 展开运算符

其用法和数组里的使用是一样的

复制

代码示例 👉

<script>
    let obj1 = {
        name: "hello"
    }

    let obj2 = {
        ...obj1
    }

    console.log(obj2);
</script>

---控制台---
Object
name: "hello"
[[Prototype]]: object

合并：

代码示例 👉

<script>
    let obj1 = {
        way: 'hello,'
    }

    let obj2 = {
        name: '小闽'
    }

    let obj3 = { ...obj1, obj2 }

    console.log(obj3);
</script>

注意：如果在合并的过程中，一旦遇到了同名属性时，后边的会把前面的属性值给覆盖掉。

Object.assign方法

<script>
    let obj1 = {
        name: 'Alian'
    }

    let obj2 = {
        age: 18
    }

    let conbainObj = Object.assign(obj1, obj2);

    console.log(conbainObj);
</script>

---控制台---
object
age: 18
name: "Alian"
[[Prototype]]: object

这和上面的扩展运算符是可以实现一样的功能的，但是其影响了Object.assign(obj1, obj2) 中第一个参数 obj1

console.log(obj1);

---控制台---
object
age: 18
name: "Alian"
[[Prototype]]: object

所以咱可以在外面声明一个空数组来借助这个合并的对象 👉

<script>
    let obj = {}

    let obj1 = {
        name: 'Alian'
    }

    let obj2 = {
        age: 18
    }

    let conbainObj = Object.assign(obj, obj1, obj2);

    console.log(conbainObj);

    console.log(obj1);
</script>

Object.is 方法

这个方法解决了JS以前的一个痛点，判断两个值是否相等，以前的 “==” 和“===” 无法判断 NaN 是否相等。这个Object.is属于是和这个“===”几乎一致

<script>
    console.log(Object.is(5, 5)); // true

    console.log(Object.is(5, "5"));  // false

    // === 和 Object.is 同样都是判断不了两个对象是否相等

    console.log(Object.is({}, {})) // false

    console.log({} === {})  // false
</script>

唯一的两点不同就是NaN的判断和+0与-0的判断

代码示例 👉

NaN

使用场景：当咱接受到两个处理过的变量，就可以使用Object.is来进行判断，进行判断传过来的是不是一个合法的数据

代码示例 👉

<script>
    console.log(NaN === NaN); // false

    console.log(Object.is(NaN, NaN));  // true
</script>

Object.is 判断 +0 和 -0 是相等的，而 === 判断 +0 和 -0是不相等的

代码示例 👉

<script>
    console.log(+0 === -0);  // false
    console.log(Object.is(+0, -0));  // true
</script>
