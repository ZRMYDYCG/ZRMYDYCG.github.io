ES6 类

类的由来

JavaScript 语言中,生成实例对象的传统方法是通过构造函数，咱来看看这个例子: 

<script>
    function Point(x, y) {
        // 属性挂载到this上面
        this.x = x;
        this.y = y;
    }

    // 方法挂载到原型上面
    let string = null
    Point.prototype.toString = function () {
        string = `${this.x}和${this.y}是一对好朋友`;
    }

    let Point1 = new Point(1, 2);

    Point1.toString()

    console.log(string)
</script>

控制台:

追溯一段历史:

上面的这种写法的确是在面向对象,但是其和传统的面向对象语言(Java/C++)的差异很大,很容易让人觉得这凭什么啊,所以设计JavaScript的老头子们,希望这个JavaScript也可以更加接近传统语言,所以引入（Class）类的概念,但由于很多原因,受到外界的很多反对和质疑,所以迟迟没有推出,直到ES6,就真正把这个Clas（类）这个概念引入啦。

JavaScript璀璨的今天:

ES6 提供了更接近传统语言的写法，引入了 Class（类）这个概念，作为对象的模板。通过class关键字，可以定义类。<

基本上，ES6 的class可以看作只是一个语法糖，它的绝大部分功能，ES5 都可以做到，新的class写法只是让对象原型的写法更加清晰、更像面向对象编程的语法而已。

上面的代码用 ES6 的class改写，就是下面这样

<script>
    let string = null
    class Point {
      // 属性挂载到 constructor 上
        constructor(x, y) {
            this.x = x;
            this.y = y;
        }
        toString() {
            string = `${this.x}和${this.y}是一对好朋友`;
        }
    }

    let Point1 = new Point(1, 2);

    Point1.toString();

    console.log(string);  // 1和2是一对好朋友
</script>

上面的代码就定义了一个 “类” ,咱可以看到里面是有一个 constructor() 方法,这玩意就是构造方法, 这个方式是特定的，名字不能修改,必须叫 constructor,而 this 关键字代表的则是实例对象 Point1。这种新的 Class 写法,其本质与咱开头写的 ES5 的构造函数 Point 是一致的。

咱再看,这个 Point 类除了构造方法,它还定义了一个 toString() 方法。但是这个定义toString()方法的写法要注意,他不可以用ES5函数的完整写法,也就是前面不需要加上 function 这个关键字, 直接把函数定义了放进去就可以了,另外,方法和方法之间不需要逗号分隔,加了会报错。

因此

ES6 的类,完全可以看作构造函数的另一种写法。

<script>
    class PointA{
        // ...
    }

    function PointB() {
        // ...
    }
    console.log(typeof PointA,typeof PointB);
    console.log(PointA === PointA.prototype.constructor)
    console.log(PointB === PointB.prototype.constructor)
</script>

上面代码表明，类的数据类型就是函数，类本身就指向构造函数。

使用的时候，也是直接对类使用new命令，跟构造函数的用法完全一致。

class Bar {
  doStuff() {
    console.log('stuff');
  }
}
const b = new Bar();
b.doStuff() // "stuff"

通过上面代码,构造函数的prototype属性，在 ES6 的“类”上面继续存在。事实上，类的所有方法都定义在类的原型对象prototype属性上面,因此我们能够看到 new 出来的实例对象可以使用calss类里面定义的方法

class Point {
  constructor() {
    // ...
  }

  toString() {
    // ...
  }

  toValue() {
    // ...
  }
}

// 等同于  所以上面就是语法糖的好处 

Point.prototype = {
  constructor() {},
  toString() {},
  toValue() {},
};

上面代码中，constructor()、toString()、toValue()这三个方法，其实都是定义在Point.prototype（原型对象）上面。

因此: 在类的实例上调用方法,其实就是调用原型对象上的方法。

class B {}
const b = new B();

b.constructor === B.prototype.constructor // true

上面代码中，b是B类的实例对象，它的constructor()方法就是B类原型的constructor()方法。

由于类的方法都定义在prototype对象上面，所以类的新方法可以添加在prototype对象上面,但是当我们采用赋值方法一次添加多个新方法时,这样修改后的原型对象 conctructor 就不再指向当前构造函数了,此时,我们可以在修改后的原型对象中,添加一个 constructor 指向原来的构造函数，而Object.assign()方法可以很方便地一次向类添加多个方法。

class Point {
  constructor(){
    // ...
  }
}

Object.assign(Point.prototype, {
  toString(){},
  toValue(){}
});

prototype 原型对象的 constructor 属性，直接指向“类”的本身，这与 ES5 的构造函数的行为是一致的。

Point.prototype.constructor === Point // true

另外，类的内部所有定义的方法，都是不可枚举的（non-enumerable）。

class Point {
  constructor(x, y) {
    // ...
  }

  toString() {
    // ...
  }
}

Object.keys(Point.prototype)
// []
Object.getOwnPropertyNames(Point.prototype)
// ["constructor","toString"]

上面代码中，toString()方法是Point类内部定义的方法，它是不可枚举的。这一点与 ES5 的行为不一致。

var Point = function (x, y) {
  // ...
};

Point.prototype.toString = function () {
  // ...
};

Object.keys(Point.prototype)
// ["toString"]
Object.getOwnPropertyNames(Point.prototype)
// ["constructor","toString"]

上面代码采用 ES5 的写法，toString()方法就是可枚举的。

ES6继承

ES6继承比咱以前学的原型继承的写法更加简洁

    <script>
       // 父类
        class Person {
            constructor(name, age) {
                this.name = name
                this.age = age
            }

            say() {
                console.log("hello");
            }
        }
       // 子类
       class Student extends Person {

        }

        let obj = new Student()

        console.log(obj)
    </script>

通过上面的代码可以使得 Student 继承到 Person 中的方法。

那如果咱要是想继承到父类中的属性呢?

方法还是有的

    <script>
        class Person {
            constructor(name, age) {
                this.name = name
                this.age = age
            }

            say() {
                console.log("hello");
            }
        }

        class Student extends Person {
            constructor(name, age, grade) {
                super(name, age)
                this.grade = grade
            }
        }

        let obj = new Student("kw", "12", "100")

        console.log(obj)

        obj.say()

        console.log(obj.name)
    </script>

需求:

既然可以在 class Student extends Person 中继承父类的属性和方法,那么可不可以,将上面代码父类中的say方法给覆盖掉呢?当然是可以的,并且还可以在覆盖之前使用super方法,将父类中的方法调用一次。

    <script>
        class Person {
            constructor(name, age) {
                this.name = name
                this.age = age
            }

            say() {
                console.log("hello");
            }
        }

        class Student extends Person {
            constructor(name, age, grade) {
                super(name, age)
                this.grade = grade
            }

            say() {
                super.say() // 语法糖
                console.log("覆盖")
            }
        }

        let obj = new Student("kw", "12", "100")

        obj.say()
    </script>

这语法糖可真甜,连 this 指向都不用管,直接拿到 Person 中的 say 方法就执行了。

类继承:使 Chinese 通过继承得到 Human 的方法, 再通过 super() 调用父类 继承父类的属性

<script type="text/javascript">
    // 父类
    class Human {
        constructor(name) {
            this.name = name
            this.kingdom = 'animal'
            this.color = ['yellow', 'white', 'brown', 'black']
        }
        // 补全代码
        getName() {
            return this.name
        }
    }

    // 补全代码
    // 子类
    class Chinese extends Human {
        constructor(name, age) {
            super(name)  //  super 调用父类的构造函数 这里是继承其属性
            this.age = age
        }

        getAge() {
            return this.age
        }
    }
</script>

试着打印子类实例对象里的属性和方法,发现当父类的属性赋初值后,子类可以通过 super() 调用继承其属性值,且再传入值也修改不了最初的继承值。

<script type="text/javascript">
    class Human {
        constructor(name) {
            this.name = name
            this.kingdom = 'animal'
            this.color = ['yellow', 'white', 'brown', 'black']
        }
        // 补全代码
        getName() {
            return this.name
        }
    }

    // 补全代码
    class Chinese extends Human {
        constructor(name, kingdom, age) {
            super(name, kingdom)
            this.age = age
        }

        getAge() {
            return this.age
        }
    }

    const obj = new Chinese('xiaohua', 'hello', 18)
    console.log(obj.name, obj.kingdom, obj.age, obj.getName());
</script>
