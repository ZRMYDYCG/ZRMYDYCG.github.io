let 声明变量

要看看它的好处，这就必须和 var 声明来比一比了

块级作用域（函数的那个是叫局部作用域）

以前呀,用这个 var 声明的的时候，很容易将if，for….里面的声明的变量暴露出来，成为了全局变量，从而污染了我们定义的全局变量。

var 声明

 {
    var a = 1;
 }
 console.log(a)

---控制台---
1

在控制台顺利地把这个块中的a给打印出来了

let 声明

 {
    let a = 1;
 }
  console.log(a);

---控制台---
Uncaught ReferenceError: a is not defined

萤火虫的比喻

需求

遍历ul里的li，点击其中的li，就在控制台打印该li的索引号。

代码示例 - var 👉

<body>
    <ul>
        <li>I</li>
        <li>Love</li>
        <li>Me</li>
    </ul>
    <script>
        var list = document.querySelectorAll('ul li')
        for (var i = 0; i < list.length; i++) {
            list[i].onclick = function () {
                console.log(i);
            }
        }
    </script>
</body>

---控制台---
我们可以发现：无论我们点击哪一个 li ，返回的都是 3

为什么每点击其中的 li，控制台都是打印 3 呢？

是因为这个 for循环 是一瞬间就完成了，而注册的这个点击事件是属于是一个异步的操作，此时的 i = 3 了，且这个 i 已经暴露成了一个全局变量了，所以每次点击返回的都是 3。

解决方案

那就是给每一个 li 自定义一个 index 属性，在循环的时候就把下标交给这个自定义属性

代码示例 - let 👉

<body>
    <ul>
        <li>I</li>
        <li>Love</li>
        <li>Me</li>
    </ul>
    <script>
        let list = document.querySelectorAll('ul li')
        for (let i = 0; i < list.length; i++) {
            list[i].onclick = function () {
                console.log(i);
            }
        }
    </script>
</body>

---控制台---
我们可以发现：
点击其中的 li ，就在控制台打印该 li 的索引号。

这就是 let 的能耐，在每一次循环后，这个 i 并没有被释放掉，每一次的点击事件依赖于这个 i，所以所点之处皆下标

不允许重复声明

用let重复声明之后，嘎嘎报错，而用var是可以这样滴，但是后面的值会覆盖掉前面的。

代码示例 👉

var i = 1;
var i = 2;
console.log(i);  ---控制台 2---
let you = 1; 报错: Cannot redeclare block-scoped variable 'you'
let you = 2; 报错: Cannot redeclare block-scoped variable 'you'

没有变量提升

代码示例 👉

// var 变量提升

    var a = '太酷辣！！！'
    console.log(a)
    console.log('................')

// let 变量没提升

    let a = '太酷辣！！！'
    console.log(a)

---控制台---
太酷辣！！！
................
Uncaught SyntaxError:Identifier 'a' has already been declared

暂存性死区

代码示例 👉

    let a = '太酷辣'
    function trys() {
        console.log(a);
    }
    trys()

---控制台---
太酷辣

    let a = '太酷辣'
    function trys() {
        console.log(a);
        let a = '煞笔煞笔';
    }
    trys()

---控制台---
Uncaught ReferenceError:Cannot access before initialization at trys

由于这个暂时性死区的存在，也就是指函数内部，声明的这个变量a是已经存在的了，但是在这声明以上的区域没法使用，要是把打印函数写到函数内部的变量声明之后就可以了。

不与顶层对象挂钩

代码示例 👉

    var a = 100;
    console.log(window.a); ---控制台 100---

    let a = 11;
    console.log(window.a) ---控制台 undefined---

const声明常量

比如一些文件路径，网页地址啥的死信息就可以使用const来定义

  1. 定义的是一个常量，不可以修改

  2. 不能重复定义

  3. 也存在块级

  4. 变量不提升

  5. 也存在暂时性死区

  6. 不与顶层对象挂钩

其实和 let 声明的特性差不多吧

但是用const声明的变量就一定不可以被修改了了吗？

代码示例 👉

    const obj = {
        name: '小黑',
        age: 18
    }
    obj.name = '999'

    const arr = [{
        age: 18
    },
    {
        age: 19
    },
    {
        age: 20
    }]

    arr[2].age = 30

---控制台---
>obj
{ name : '999', age: 18}
[[Prototype]]: Object
> arr
0: {age: 18}
1: {age: 19}
2: {age: 30}
length: 3
[[Prototype]]: Array(0)

当const声明的是复杂数据类型的时候，如上例，其声明的obj和arr，实际上是指向对象和数组的一个指针(地址)，通过该指针找到该对象中某一属性将其修改。

但是这里有一种 freeze 方法可以冻结对象中的属性,使其不能被修改

代码示例 👉

    const obj = Object.freeze({
        name: '小黑',
        age: 18
    })
    obj.age = '19'
    console.log(obj.name)
