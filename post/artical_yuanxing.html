构造函数

封装是面向对象思想中比较重要的一部分,js面向对象可以通过构造函数实现封装,构造函数方法虽然好用,但是存在浪费内存问题。

<script>
    function Star(name, age) {
        this.name = name
        this.age = age
        this.sing = function () {
            console.log("唱歌");
        }
    }
    const p1 = new Star("刘德华", 16)
    const p2 = new Star("张学友", 19)

    console.log(p1.sing === p2.sing) // 返回 false
</script>

上面代码每当我们 new 一个实例对象, 都会开辟一个新的内存空间来存放 sing 方法, sing 方法不像上面的简单数据类型 name age ,简单数据类型里面存的是值,不用就销毁掉了,是没啥问题的。

原型对象

  • 构造函数通过原型分配的函数是所有对象所共享的

  • JavaScript 规定,每一个构造函数都有一个 prototype 属性, 指向另一个对象,而prototype又是以一个对象来表示,所以我们也称其为原型对象。

console.dir(Star.prototype)

  • 这个对象可以挂载函数,对象实例化不会多次创建原型上的函数,也就是说函数就在原型对象上面了,实例对象直接调用就可以了,这样就节约了内存。

  • 我们可以把那些不变的方法, 直接定义在 prototype 对象上,这样所有实例对象就可以共享这些方法。

<script>
    function Star(name, age) {
        this.name = name
        this.age = age
    }

    Star.prototype.sing = function () {
        console.log("唱歌");
    }

    const p1 = new Star("刘德华", 16)
    const p2 = new Star("张学友", 19)

    p1.sing()
    p2.sing()

    console.log(p1.sing === p2.sing)
</script>

通过控制台的打印,我们会发现,返回的是 true, 而最开始的代码里面打印的 sing 方法是 false ,这也进一步说明了,原型对像确实是实现了方法的共享,节约了内存。

  • 构造函数和原型对象中的 this 都指向 实例化的对象
<script>
    let that
    function Star() {
        that = this
        this.name = name
    }


    // 实例对像 p1
    const p1 = new Star("刘得华")
    console.log(that = p1); // 返回true
</script>
<script>
    let that
    function Star() {
        this.name = name
    }

    Star.prototype.sing = function () {
        that = this
        console.log("嫦娥")
    }
    // 实例对像 p1
    const p1 = new Star("刘得华")
    p1.sing()
    console.log(that === p1); // 返回true
</script>

constructor 属性及其应用

construct 单词意思: 构造函数

那么这个属性在哪呢?

<script>
    function Star() {

    }

    const p1 = new Star()

    console.log(Star.prototype);
</script>

通过上面代码及控制台的打印,我们发现:这个constructor属性在原型对像身上,其实每个原型对像身上都有一个constructor属性 (constructor 构造函数)

那么它的作用是什么呢?

该属性指向该原型对像的构造函数,简单理解,就是指向我的爸爸,我是有爸爸的孩子.

<script>
    function Star() {

    }

    const p1 = new Star()

    console.log(Star.prototype.constructor === Star); // true
</script>

通过上面的打印,返回值为 true ,也说明了该原型对象上的 constructor 属性指回了构造函数,也就是说这个 constructor 属性能找到创造这个原型对象的构造函数.

那么它的使用场景是什么呢? 咱先来看一个小对比 

<script>
    function Star1() {

    }

    Star1.prototype.sing = function () {
        console.log("歌");
    }

    Star1.prototype.dance = function () {
        console.log("舞");
    }
    console.log(Star1.prototype)

    function Star2() {

    }
    Star2.prototype = {
        sing: function () {
            console.log("歌");
        },
        dance: function () {
            console.log("舞");
        }
    }
    console.log(Star2.prototype)
</script>

咱通过控制台的打印可以看到,当原型对像上有多个方法时,其实也就是做了一个赋值操作时(Star2.prototype),它这个原型对像就不知道指向哪个构造函数的了

也就是说:如果有多个对像的方法,我们可以给原型对象采取对象形式赋值,但是这样会覆盖构造函数原型对象原来的内容,这样修改后的原型对象 conctructor 就不再指向当前构造函数了,此时,我们可以在修改后的原型对象中,添加一个 constructor 指向原来的构造函数

所以有如下解决方法:

重新指回它的构造函数

   Star2.prototype = {
        constructor: Star2,
        sing: function () {
            console.log("歌");
        },
        dance: function () {
            console.log("舞");
        }
    }
    console.log(Star2.prototype)

对象原型 _ _ prototype _ _

思考:

构造函数可以创建实例对象,构造函数还有一个原型对象,一些公共的属性或者方法放到这个原型对象身上,但是为啥实例对象可以访问原型对象里面的属性和方法呢?

之所以实例对象可以访问到原型对象上的属性和方法,是因为实例对象身上有一个对象原型(_ prototype _)会指向原型对象

<script>
    let Obj1 = new Object()
    Obj1.name = "2222"

    let Obj2 = {
        name: "111"
    }

    function Star() {

    }

    const p1 = new Star()

    console.log(Obj1)
    console.log("-----------")
    console.log(Obj2)
    console.log("-----------")
    console.log(p1)
</script>

通过上面的控制台打印,我们可以看到每一个对象都有一个对象原型_ prototype _ ,对象原型里都有一个 constructor 属性

<script>
    function Star() {

    }

    const p1 = new Star()

    console.log(p1.__proto__)

    // 对象原型__proto__指向 该构造函数的原型对象
    console.log(p1.__proto__ === Star.prototype)

    // 对象原型里面有constructor 指向 构造函数 Star
    console.log(p1.__proto__.constructor === Star)
</script>

通过控制台的打印,验证成功!!!

注意:

  • _ _ prototype _ _ 是JS非标准属性

  • [[ prototype ]] 和 _ _ prototype _ _意义相同

  • 用来表明当前实例对象指向哪个原型对象 prototype

  • _ _ prototype _ _ 对象原型里面也有一个 constructor 属性,指向创建该实例对象的构造函数

  • 对象都会有一个属性对象原型 _ _ proto _ _ 指向构造函数的 prototype 原型对象,之所以我们对象可以使用构造函数 prototype 原型对象的属性和方法,就是因为对象有_ _proto _ _ 对象原型的存在

原型继承

继承是面向对象编程的另一个特征,通过继承进一步提升代码封装的程度,JavaScript 中大多是借助原型对象实现继承的特性

咱来看个代码:

<script>
    // 女人
    function Woman() {
        this.eays = 2
        this.head = 1
    }
    const W1 = new Woman()
    // 男人
    function Man() {
        this.eays = 2
        this.head = 1
    }
    const M1 = new Man()
</script>

上面代码中男人女人都有共同的属性 eays 和 head,那咱可不可以将其抽取出来

<script>
    // 抽取
    let Person = { eays: 2, head: 1 }
    // 女人  想要继承 Person
    function Woman() { }
    Woman.prototype = Person
    const W1 = new Woman()
    // 男人 想要继承 Person
    function Man() { }
    Man.prototype = Person
    const M1 = new Man()
    // Woman Man 通过原型来继承 Person
    console.log(W1)
    console.log(M1)
</script>

通过控制台打印,我们会发现这种原型继承的方式,同样可以使得实例对象通过对象原型(_ _ proto_ _)访问到原型对象上的属性和方法.

但是我们会发现:我们把 Person 是以赋值的形式交给了构造函数的原型对象,会覆盖掉原型对象的 constructor 属性,所以要再重新指回原来的实例对象

<script>
    // 抽取
    let Person = { eays: 2, head: 1 }
    // 女人  想要继承 Person
    function Woman() { }
    // 男人 想要继承 Person
    function Man() { }
    // Woman Man 通过原型来继承 Person
    Woman.prototype = Person
    Man.prototype = Person
    Woman.prototype.constructor = Woman
    Man.prototype.constructor = Man
    const M1 = new Man()
    const W1 = new Woman()
    console.log(Man.prototype)
    console.log(Woman.prototype)
    console.log(M1)
    console.log(W1)
</script>

注意: 原型继承的一个问题

比如我们单独给 Woman 构造函数的原型对象上添加一个方法

<script>
    let Person = { eays: 2, head: 1 }
    function Woman() { }
    function Man() { }
    Woman.prototype = Person
    Woman.prototype.constructor = Woman
    Woman.prototype.baby = function () {
        console.log("生宝宝")
    }
    Man.prototype = Person
    Man.prototype.constructor = Man
    const M1 = new Man()
    const W1 = new Woman()
    console.log(M1)
    console.log(W1)
</script>

通过打印台,我们会发现:M1这个在其原型对象身上也有了baby这个方法

这是因为Woman 和 Man 的原型对象是一模一样的,都接收了同一个对象Person ,根据引用类型的特点,他们指向同一个对象,修改一个就会都影响

其实可以通过构造函数的方式来解决这个问题

<script>
    function Person() {
        this.eyes = 2
        this.head = 1
    }

    function Women() {

    }

    Women.prototype = new Person()

    Women.prototype.baby = function () {
        console.log("生宝宝")
    }

    const W1 = new Women()

    function Man() {

    }

    Man.prototype = new Person()

    const M1 = new Man()

    console.log(W1)
    console.log(M1)
</script>

上面是用原型对象上添加了的是 new 出来的实例对象,我们知道实例对象间是互不影响的

原型链

基于原型对象的继承使得不同构造函数的原型对象关联在一起,并且这种关联的关系是一种链状的结构,我们将原型对象的链状结构称为原型链

通过上面的关系,咱不妨总结条规两条律:

  • 只要是原型对象,就有 constructor 属性,指回创造这个原型对象的构造函数

  • 只要是对象,就有 _ _ proto _ _ ,指向原型对象

原型链 – 其实是一种查找规则

查找顺序如下:

  1. 当访问一个对象的属性(包括方法)时,首先查找这个对象自身有没有该属性.

  2. 如果没有,查找到它的原型对象(也就是_ _ proto _ _ 指向的 prototype 原型对象)

  3. 如果还没有,查找到它原型对象的原型(Object 的原型对象)

  4. 依此类推一直找到 Object 为止,找不到就为 null

  5. _ _ proto _ _ 对象原型的意义就在于为对象成员查找机制提供一个方向,或者说一条路线

  6. 可以使用 instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上

当然本图中是拿了 Object 构造函数来举例,其实 Array 构造函数….也是如此,所以我们在查阅文档的时候 比如这个map方法,文档里是这样写的 Array.prototype.map(),实际上就说明了这个map方法是放在了Array 构造函数的原型对象身上,所有咱直接 数组名.map() 就可以使用这个map方法了,实际上它也是遵循了这个原型链的查找机制,正是因为有这个_ _ proto _ _ 的存在,所以可以新创建出来的 数组和对象就可以使用到原型对象上的这些方法.

instanceof 运算符

运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上

<script>
    function Person() {

    }

    const p1 = new Person()

    console.log(p1 instanceof Person) // true
    console.log(p1 instanceof Object) // true
    console.log(p1 instanceof Array) // false
</script>

                    this.oBoxItems[index].classList.add("active")
