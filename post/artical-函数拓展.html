函数拓展

参数默认值

这个参数默认值是适用于任何的函数场景的，是很不错的一种实际开发项目的方法和技巧。

比如现在封装了一个ajax的方法

<script>
    function ajax(url, methods, async) {
        console.log(url, methods, async);
    }

    ajax("//http:baidu.com", "get", true)
</script>

那么这个参数默认值该怎么用呢？其实很简单，就是给函数参数赋上初值，若调用函数时没有传这个参数，那就会使用默认值，如果函数调用传入参数，就会更新初值。

<script>
    function ajax(url, methods = "get", async = true) {
        console.log(url, methods, async);
    }

    ajax("//http:baidu.com", true)
    ajax("//http:baidu.com", "get", false)
</script>

---控制台---
//http:baidu.com true true
//http:baidu.com get false

… 展开运算符的运用

比如现在咱想要得到函数调用传入的值形成数组，可以使用 arguments 和 Array.from 方法

代码示例 👉

<script>
    function f1() {
        let arr = Array.from(arguments)
        console.log(arr);
    }

    f1(1, 2, 3, 4, 5, 6)
</script>

---控制台---
Array(6) [1,2,3,4,5,6]

使用 ...

这拿到的是第一个值 1

    function f1(data) {
        console.log(data);
    }

    f1(1, 2, 3, 4, 5, 6)
</script>

这便拿到了它全家了 并形成数组 [1,2,3,4,5,6]

<script>
    function f1(...data) {
        console.log(data);
    }

    f1(1, 2, 3, 4, 5, 6)
</script>

再看：那咱现在不要1，2，3 就要4，5，6 那咋整？

<script>
    function f1(x, y, z, ...data) {
        console.log(data);
    }

    f1(1, 2, 3, 4, 5, 6)
</script>

name属性

就是能够拿到函数的名字

<script>
    function f1(x, y, z, ...data) {
        console.log(data);
    }
    console.log(f1.name) // output: f1
    f1(1, 2, 3, 4, 5, 6) // output: [4,5,6]
</script>

箭头函数

最好的称赞往往只需要最简单的文字：牛逼

咱以前的函数写法有:

  • 声明式

  • 赋值式

<script>
    // 声明式
    function f1() {

    }

    // 赋值式
    let f2 = function () {

    }
</script>

那这个箭头函数咋写呢？

把function删掉，小括弧后边加箭头

咱先看看这个箭头函数最初始的写法 👉

<script>
    let f2 = () => {

    }
</script>

那看这最初始的写法，好像是还看不出它的简洁性，咱继续看看

如果箭头函数体内只有一个返回值，而没有其他逻辑，就可以把花括弧 { } 和 return 都省略掉

<script>
    let f2 = () => "箭头函数！！！！"
    console.log(f2())
</script>

---控制台---
箭头函数！！！！

咱再写一个渲染列表的案例来好好体会箭头函数的简洁性

代码示例 👉

这是普通写法

<ul></ul>
<script>
    let arr = ["箭头", "函数", "真", "简洁"]
    let newArr = arr.map(function (item, index) {
        return `<li data-id="${index}">${item}</li>`
    })
    console.log(newArr);
    document.querySelector('ul').innerHTML = newArr.join("")
</script>

箭头简写

function是不是可以删掉，函数体内是不是只有一个return而无其他逻辑，是不是可以把{ }去掉

<ul></ul>
<script>
    let arr = ["箭头", "函数", "真", "简洁"]
    let newArr = arr.map((item, index) => `<li data-id="${index}">${item}</li>`)
    console.log(newArr)
    document.querySelector('ul').innerHTML = newArr.join("")
</script>

注意：如果 return 返回的是一个对象，也不是不可以省略这个return,就是这个对象要加上一个括弧

代码示例 👉

<script>
    let f3 = () => ({
        name: '箭头函数真好用！！！'
    })
</script>

那能不能再简单一点，当然可以呀，当咱只传入一个参数的时候，可以把小括弧也打烂来。

代码示例 👉

<script>
    let arr = [1, 2, 3]
    let newArr = arr.map(item => `<li>${item}</li>`)
    console.log(newArr)
</script>

箭头函数所带来的问题

  1. 无法访问数组内建方法 arguments
<script>
    let judge = () => {
        console.log(arguments)
    }

    judge(1, 2, 3, 4, 5, 6)
</script>

---控制台---
Uncaught ReferenceError : arguments is not definet at judge
  1. 无法 new（）一个实例对象
<script>
    let judge = () => {

    }
    new judge()
</script>

---控制台---
Uncaught TypeError: judge is not a constructor
  1. 箭头函数没有 this，或者还有一种说法是箭头函数的 this 是指向父作用域

但是就是这样的一个问题，可以带来很大的价值。

咱先看看普通函数的一个痛点 👉

你倒是敲代码啊: <input type="text">
<script>
    let input = document.querySelector('input')
    input.oninput = function () {
        console.log(this);  // 指向 input
        setTimeout(function () {
            console.log(this)  // 指向 windows
            console.log(`我收到了用户的数据,是${this.value}`)
        }, 1000)
    }
</script>

---控制台---
<input type="text">

Window {window: Window, self: Window, document: document,name:''Location:Location, ....}

我收到了用户的数据,是undefined

解决方法 👉

声明了一个临时 this（_this）变量存放 input

你倒是敲代码啊: <input type="text">
<script>
    let input = document.querySelector('input')
    input.oninput = function () {
        let _this = this
        setTimeout(function () {
            console.log(`我收到了用户的数据,是${_this.value}`)
        }, 1000)
    }
</script>

好了，箭头函数是不是就可以轻松解决掉这个问题？

你倒是敲代码啊: <input type="text">
<script>
    let input = document.querySelector('input')
    input.oninput = () => {
        setTimeout(() => {
            console.log(`我收到了用户的数据,是${this.value}`)
        }, 1000)
    }
</script>

---控制台---
window:{ window, self: window, document: document, name: '',location: Location,...}

我收到了用户的数据,是 undefined

哈哈，咱这一下改爽了，给外边注册事件的函数也用了箭头函数，本来是指向 input 的，结果指到外层的 windows 了，那么定时器函数里的 this 就指向了外层函数的指向了，也即是windiows

像这种注册事件函数最好是写成普通函数形式

其内部的异步操作可以写箭头函数 👉

你倒是敲代码啊: <input type="text">
<script>
    let input = document.querySelector('input')
    input.oninput = function () {
        console.log(this)
        setTimeout(() => {
            console.log(`我收到了用户的数据,是${this.value}`)
        }, 1000)
    }
</script>

---控制台---
<input type="text'>
我收到了用户的数据，是1
