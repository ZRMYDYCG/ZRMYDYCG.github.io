Promise 的状态

实例对象中的一个属性 【PromiseState】

  • pending 未决定的

  • resolved / fullfilled 成功

  • rejected 失败

这是 Promise 实例对象的一个内置属性

其变化方式:

  • pending 变为 resolved

  • pending 变为 rejected

说明:

只有这 2 种变化,且一个 promise 对象只能改变一次

无论变成成功还是失败,都会有一个结果数据

成功的结果数据一般称为 value, 失败的结果数据一般称为 reason

Promise 对象的值

实例对象中的另一个属性 【PromiseResult】
保存的是异步任务【成功/失败】的结果

  • resolve

  • reject

Promise 工作流程

先创建一个 Promise 对象,在 Promise 对象内部封装异步操作,如果异步操作成功,则调用 resolve 函数,resolve 函数一调,则将 Promise 函数状态改为 resolved（成功）,因此在之后调用 then 方法时将调用的是第一个参数（也即是第一个回调函数中的代码）,然后返回一个新的 Promise 对象（then 方法返回的 Promise 对象）。

Promise 相关的 API

  1. Promise 的构造函数:Promise（excutor）{ }

  • executor 函数: 执行器（resolve , reject）=> { }

  • resolve 函数: 内部定义成功时我们调用的函数 value =>{ }

  • reject 函数: 内部定义失败时我们调用的函数 reson =>{ }

说明: executor 会在 Promise 内部立即同步调用,异步操作在执行器中执行。

  1. Promise.prototype.then 方法:（onRseolved,onRejected）=> { }

  • onResolved 函数: 成功的回调函数 (value) => { }

  • onRejected 函数: 失败的回调函数 (reason) => { }

说明: 指定用于得到成功 value 的成功回调函数和用于得到失败 reason 的失败回调函数返回一个新的 promise 对象。

  1. Promise.prototype.catch 方法: (onRejected) => { }

  • onReject 函数: 失败的回调函数 (reason) => { }

  • catch 方法只能指定失败的回调

<script>
    const p = new Promise((resolve, reject) => {
        reject('catch 方法只在失败时执行回调')
    });

    p.catch(reason => {
        console.log(reason);
    });
</script>
  1. Promise.resolve 方法 : (value) => { }

  • 这个方法比较特殊,它是属于这个 Promise 方法函数的对象,并不属于 Promise 实例对象

  • value: 成功的数据或 promise 对象
    （说明: 返回一个成功/失败的 promise 对象）

  • 如果传入的参数为 非 Peomise 类型的对象,则返回的结果为成功 promise 对象

<script>
    const p1 = Promise.resolve(521);
    console.log(p1);
</script>
  • 如果传入的参数为 Promise 类型的对象,则参数的结果决定了 resolve 的结果
<script>
    const p2 = Promise.resolve(new Promise((resolve, reject) => {
        resolve('ok');
    }));

    console.log(p2);

    const p3 = Promise.resolve(new Promise((resolve, reject) => {
        reject('Error')
    }));
    
    console.log(p3);

    p3.catch(reason => {
        console.log(reason);
    });
</script>
  1. Promise.reject 方法: (reason) => { }

  • 它是属于 Promise 函数对象的方法,不是属于 Promise 实例对象的方法

  • reason: 失败的原因
    （说明: 返回一个失败的 promise 对象）

<script>
    let p = Promise.reject(521)
    let p1 = Promise.reject(new Promise((resolve, reject) => {
        resolve('OK');
    }));
    console.log(p);
    console.log(p1);
</script>
  1. Promise.all 方法: (promise) => { }

  • Promise 函数对象的一个方法, 不是 Promise 实例对象的方法

  • promise: 包含 n 个 promise 数组，也就是每一个都是 promise 对象
    （说明: 返回一个新的 promise,只有所有的 promise 都成功时才成功,只要有一个失败了就直接失败）

<script>
    // 设置 3 个成功的 Promise 对象
    let p1 = new Promise((resolve, reject) => {
        resolve('OK');
    });

    let p2 = Promise.resolve('success');

    let p3 = Promise.resolve('Pass');

    let result = Promise.all([p1, p2, p3]);

    console.log(result);
</script>

从上面代码和控制台的打印结果,我们可以看到 all 方法会返回一个 promise 对象,只有当 p1,p2,p3 都成功的情况下,才会返回一个成功的 promise 对象,其结果值为 p1 p2 p3 这三个 promise 对象成功的结果组成的数组。

现在咱将其中一个设置为失败

<script>
    let p1 = new Promise((resolve, reject) => {
        resolve('OK');
    });

    let p2 = Promise.reject('Error');

    let p3 = Promise.resolve('Pass');

    let result = Promise.all([p1, p2, p3]);

    console.log(result);
</script>

从上面代码和控制台的打印结果,我们可以看到 all 方法会返回一个 promise 对象,当 p1 p2 p3 中不全成功的情况下,会返回一个失败的 promise 对象,其结果值为失败的那个promise值的结果值。

  1. Promise.race 方法: (promise) => { }
  • promise: 包含 n 个 promise 的数组
    （说明:返回一个新的 promise 对象, 第一个完成的 promise 的结果状态就是最终的结果状态）
<script>
    let p1 = new Promise((resolve, reject) => {
        resolve('OK');
    });

    let p2 = Promise.reject('Error');

    let p3 = Promise.resolve('Pass');

    let result = Promise.all([p1, p2, p3]);

    console.log(result);
</script>

由于是 p1 先改变状态,所以 result 的状态和值就是由 p1 来决定。

<script>
    let p1 = new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve('OK');
        }, 1000)
    });

    let p2 = Promise.reject('Error');

    let p3 = Promise.resolve('Pass');

    let result = Promise.all([p1, p2, p3]);

    console.log(result);
</script>

这里 p2 是最先改变状态的

Promise 的几个关键问题

  1. 如何改变 promise 的状态

(a) 调用 resolve 函数 pending -> fulfilled

<script>
    let p = new Promise((resolve, reject) => {
        resolve('OK');
    });
    console.log(p);
</script>

(b) 调用 reject 函数 pending -> rejected

<script>
    let p = new Promise((resolve, reject) => {
        reject('Error');
    });
    console.log(p);
</script>

(c) throw 抛出错误 pending -> rejected

<script>
    let p = new Promise((resolve, reject) => {
        throw '出问题了';
    });
    console.log(p);
</script>
  1. 一个 promise 指定多个成功/失败回调函数,都会调用吗?

当 promise 改变为对应状态时都会调用。

<script>
    let p = new Promise((resolve, reject) => {
        resolve('OK');
    });

    // 指定回调 - 1

    p.then(value => {
        console.log(value);
    });

    // 指定回调 - 2

    p.then(value => {
        alert(value);
    });
</script>
  1. 改变promise状态 和 指定回调函数(注意: 指定不是执行) 谁先谁后?

这里的谁先谁后指的是执行的谁先谁后,也就可以简化理解为是promise状态改变先执行还是then方法指定回调先执行

(a) 都有可能,正常情况下是先指定回调再改变状态,但也可以先改变状态再指定回调。

<script>
    let p = new Promise((resolve, reject) => {
        setInterval(() => {
            resolve('OK')
        }, 1000);
    });
    p.then(value => {

    }, reason => {

    });
</script>

(b) 如何先改变状态再指定回调?

  • 在执行器中直接调用 resolve() / reject(),也即是同步任务
<script>
    let p = new Promise((resolve, reject) => {
        resolve('OK')
    });
    p.then(value => {

    }, reason => {

    });
</script>
  • 延长更长的时间调用 then
<script>
    let p = new Promise((resolve, reject) => {
        setInterval(() => {
            resolve('OK');
        }, 1000);
    });

    setInterval(() => {
        p.then(value => {

        }, reason => {

        });
    }, 2000)
</script>

(c) 什么时候才能得到数据?

其实也就是指 这个 then 指定的回调什么时候执行,其实关键的是先指定了回调,后改变状态,必须状态改变后才进入异步队列执行回调函数。
例:

<script>
    let p = new Promise((resolve, reject) => {
        setInterval(() => {
            resolve('OK')
        }, 1000);
    });
    p.then(value => {

    }, reason => {

    });
</script>
  1. promise.then() 返回的新 promise 的结果状态由什么决定?

(a) 简单表达:由 then() 指定的回调函数执行的结果决定

(b) 详细表达:

  • 如果抛出异常,新 promise 变为 rejected,reason 为抛出异常

  • 如果返回的是非 promise 的任意值,新 promise 变为resloved,value 为返回的值

  • 如果返回的是另一个新 promise,此 promise 的结果就会成为新 promise 的结果

  1. promise 如何串连多个操作任务?

(a) promise 的 then() 返回一个新的promise,可以开成then()的链式调用

(b) 通过 then() 的链式调用串连多个同步/异步任务
